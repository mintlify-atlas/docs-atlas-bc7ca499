## Overview

EasyACT uses [Pinia](https://pinia.vuejs.org/) for centralized state management. Pinia is the official state management library for Vue 3, offering a simpler and more intuitive API than Vuex.

<Info>
  Pinia is installed in `main.js` and available globally throughout the application.
</Info>

## Store Architecture

EasyACT currently has one main store:

```
src/stores/
└── accountStore.js    # Manages all financial data and API calls
```

## Account Store

The `accountStore` is the central hub for all financial data, managing transactions, categories, and actions.

**Location**: `src/stores/accountStore.js`

### Store Structure

<CodeGroup>
```javascript State
import { defineStore } from 'pinia'
import axios from 'axios'

const API_BASE_URL = 'http://localhost:8080/api'

export const useAccountStore = defineStore('account', {
  state: () => ({
    accountBooks: [],      // Array of all transactions
    isLoading: true,       // Loading state
    error: null,          // Error message
  }),
  // ...
})
```

```javascript Getters
getters: {
  // Filter income transactions
  incomeOnly: (state) => {
    return state.accountBooks.filter((book) => book.actionId === 'Income')
  },
  
  // Filter expense transactions
  expensesOnly: (state) => {
    return state.accountBooks.filter((book) => book.actionId === 'Expenses')
  },
  
  // Calculate total amount
  totalAmount: (state) => {
    return state.accountBooks.reduce((sum, item) => sum + item.amount, 0)
  },
  
  // Get records by month (with parameter)
  getRecordsByMonth: (state) => {
    return (yearMonthString) => {
      if (!yearMonthString) return state.accountBooks
      return state.accountBooks.filter((book) => {
        return book.date && book.date.startsWith(yearMonthString)
      })
    }
  },
}
```

```javascript Actions
actions: {
  // Check backend server status
  async checkServerStatus() {
    try {
      await axios.get(`${API_BASE_URL}/health`)
      console.log('後端連線正常')
      this.isServerOnline = true
    } catch (error) {
      console.error('無法連線到後端，請檢查伺服器是否啟動')
      this.isServerOnline = false
    }
  },
  
  // Fetch all account books
  async fetchAccountBooks() {
    this.isLoading = true
    this.error = null
    try {
      const response = await axios.get(`${API_BASE_URL}/account-books`)
      this.accountBooks = response.data
    } catch (err) {
      this.error = '無法連線到伺服器'
    } finally {
      this.isLoading = false
    }
  },
  // ... more actions
}
```
</CodeGroup>

<ResponseField name="accountBooks" type="Array" default="[]">

<ResponseField name="isLoading" type="Boolean" default="true">

### Getters

Getters are computed properties that derive state:

<ParamField path="incomeOnly" type="Array">
  Returns only transactions with `actionId === 'Income'`
</ParamField>

<ParamField path="expensesOnly" type="Array">
  Returns only transactions with `actionId === 'Expenses'`
</ParamField>

<ParamField path="totalAmount" type="Number">

<ParamField path="getRecordsByMonth" type="Function">

### Actions

#### Server Health Check

```javascript
await store.checkServerStatus()
```

Checks if the backend server is online by calling the `/api/health` endpoint.

#### Fetch Account Books

```javascript
await store.fetchAccountBooks()
```

Fetches all transactions from the backend and updates `accountBooks` state.

<Note>
  This action sets `isLoading` to `true` at the start and `false` when complete. It also handles errors by setting the `error` state.
</Note>

#### Add Account Book

```javascript
const success = await store.addAccountBook({
  date: '2024-03-03T10:30:00',
  amount: 1500,
  description: '午餐',
  actionId: 'Expenses',
  categoryId: 'Food'
})
```

**Parameters**:
- `newRecord` (Object): Transaction record to create

**Returns**: `Boolean` - `true` if successful, `false` otherwise

#### Update Account Book

```javascript
const success = await store.updateAccountBook(recordId, {
  date: '2024-03-03T10:30:00',
  amount: 1800,
  description: '午餐 (更新)',
  actionId: 'Expenses',
  categoryId: 'Food'
})
```

**Parameters**:
- `id` (String/Number): Record ID to update
- `record` (Object): Updated transaction data

**Returns**: `Boolean` - Success status

#### Delete Account Books (Batch)

```javascript
const success = await store.deleteAccountBooks([1, 2, 3])
```

**Parameters**:
- `ids` (Array): Array of record IDs to delete

**Returns**: `Boolean` - Success status

<Warning>
  This is a batch delete operation. All specified IDs will be deleted in a single API call.
</Warning>

#### Fetch Categories

```javascript
const categories = await store.fetchCategories()
// Returns: [{ id: 'Food', name: '飲食' }, ...]
```

Fetches available expense/income categories from the backend.

**Returns**: `Array` - Array of category objects (empty array on error)

#### Fetch Actions

```javascript
const actions = await store.fetchActions()
// Returns: [{ id: 'Income', name: '收入' }, { id: 'Expenses', name: '支出' }]
```

Fetches available action types (Income/Expenses).

**Returns**: `Array` - Array of action objects (empty array on error)

#### Add Category

```javascript
const success = await store.addCategory({
  id: 'Shopping',
  name: '購物'
})
```

**Parameters**:
- `newCategory` (Object): Category to create

**Returns**: `Boolean` - Success status

#### Add Action

```javascript
const success = await store.addAction({
  id: 'Transfer',
  name: '轉帳'
})
```

**Parameters**:
- `newAction` (Object): Action type to create

**Returns**: `Boolean` - Success status

## Using the Store in Components

### Basic Usage

<Steps>
  <Step title="Import the store">
    ```vue

    import { useAccountStore } from '@/stores/accountStore'
    </script>
    ```
  </Step>

  <Step title="Create store instance">

  </Step>

  <Step title="Access state and getters">
    ```vue
    <script setup>
    import { useAccountStore } from '@/stores/accountStore'
    
    const store = useAccountStore()
    
    // Access state
    console.log(store.accountBooks)
    console.log(store.isLoading)
    
    // Access getters
    const income = store.incomeOnly
    const expenses = store.expensesOnly
    const total = store.totalAmount
    </script>
    ```
  </Step>

  <Step title="Call actions">
    ```vue
    <script setup>
    import { onMounted } from 'vue'
    import { useAccountStore } from '@/stores/accountStore'
    
    const store = useAccountStore()
    
    onMounted(async () => {
      await store.fetchAccountBooks()
    })
    
    const addTransaction = async () => {
      const success = await store.addAccountBook({
        date: new Date().toISOString(),
        amount: 1000,
        actionId: 'Expenses',
        categoryId: 'Food'
      })
      
      if (success) {
        // Refresh data
        await store.fetchAccountBooks()
      }
    }
    </script>
    ```
  </Step>
</Steps>

### Reactivity with Pinia

<AccordionGroup>
  <Accordion title="State is automatically reactive">
    ```vue
    <template>
      <div v-if="store.isLoading">Loading...</div>
      <div v-else>
        <p>Total records: {{ store.accountBooks.length }}</p>
      </div>
    </template>
    
    <script setup>
    import { useAccountStore } from '@/stores/accountStore'
    const store = useAccountStore()
    </script>
    ```
    
    The template automatically updates when `store.isLoading` or `store.accountBooks` changes.
  </Accordion>

  <Accordion title="Using storeToRefs for destructuring">
    If you need to destructure state while maintaining reactivity:
    
    ```vue
    <script setup>
    import { storeToRefs } from 'pinia'
    import { useAccountStore } from '@/stores/accountStore'
    
    const store = useAccountStore()
    const { accountBooks, isLoading, error } = storeToRefs(store)
    
    // Now these are reactive refs
    console.log(accountBooks.value)
    </script>
    ```
    
    <Warning>
      Don't destructure actions with `storeToRefs`. Actions should be called directly on the store:
      
      ```javascript
      // ✅ Correct
      await store.fetchAccountBooks()
      
      // ❌ Wrong
      const { fetchAccountBooks } = storeToRefs(store)
      ```
    </Warning>
  </Accordion>

  <Accordion title="Computed properties with getters">
    ```vue
    <script setup>
    import { computed } from 'vue'
    import { useAccountStore } from '@/stores/accountStore'
    
    const store = useAccountStore()
    
    // Getter with parameter
    const currentMonthRecords = computed(() => {
      const yearMonth = '2024-03'
      return store.getRecordsByMonth(yearMonth)
    })
    </script>
    ```
  </Accordion>
</AccordionGroup>

## Error Handling

The store implements consistent error handling:

```javascript
async fetchAccountBooks() {
  this.isLoading = true
  this.error = null  // Reset previous errors
  
  try {
    const response = await axios.get(`${API_BASE_URL}/account-books`)
    this.accountBooks = response.data
  } catch (err) {
    this.error = '無法連線到伺服器'
    // Error is stored in state, components can display it
  } finally {
    this.isLoading = false
  }
}
```

### Handling Errors in Components

## Best Practices

<AccordionGroup>
  <Accordion title="Always handle loading states">
    ```vue
    <template>
      <div v-if="store.isLoading">Loading...</div>
      <div v-else-if="store.error">Error: {{ store.error }}</div>
      <div v-else>
        <!-- Display data -->
      </div>
    </template>
    ```
  </Accordion>

  <Accordion title="Refresh data after mutations">
    After creating, updating, or deleting records, refresh the data:
    
    ```javascript
    const success = await store.addAccountBook(newRecord)
    if (success) {
      await store.fetchAccountBooks()  // Refresh data
    }
    ```
  </Accordion>

  <Accordion title="Use getters for derived state">
    Instead of filtering data in components, use store getters:
    
    ```javascript
    // ✅ Good: Use getter
    const income = store.incomeOnly
    
    // ❌ Avoid: Filter in component
    const income = store.accountBooks.filter(b => b.actionId === 'Income')
    ```
  </Accordion>

  <Accordion title="Don't mutate state directly">
    Always use actions to modify state:
    
    ```javascript
    // ❌ Wrong: Direct mutation
    store.accountBooks.push(newRecord)
    
    // ✅ Correct: Use action
    await store.addAccountBook(newRecord)
    await store.fetchAccountBooks()
    ```
  </Accordion>
</AccordionGroup>

    ```javascript
      }),
      
      getters: {
        isLoggedIn: (state) => state.currentUser !== null,
      },
      
      actions: {
        async login(credentials) {
          // Implementation
        },
      },
    })
    ```
  </Step>

  <Step title="Use in components">
    ```vue
    <script setup>
    import { useUserStore } from '@/stores/userStore'
    ```

  </Step>
</Steps>
