## Overview

EasyACT is built using modern web technologies across frontend, backend, and database layers. This page documents all technologies, their versions, and specific use cases.

<Note>
  All version numbers are extracted from `package.json` (frontend) and `pom.xml` (backend) configuration files.
</Note>

## Frontend Technology Stack

The frontend is a Vue.js 3 application built with Vite and deployed via Nginx.

<CardGroup cols={2}>

  </Card>
  
  <Card title="Vite" icon="bolt">

  </Card>

### Routing & State Management

| Technology | Version | Purpose |
|------------|---------|----------|

## Backend Technology Stack

The backend is a Spring Boot application using Java 25 with Maven for dependency management.

### Core Framework

<Card title="Spring Boot" icon="java">
  **Version**: 4.0.1
  
  Enterprise-grade Java framework providing:
  - Auto-configuration
  - Embedded servlet container
  - Production-ready features
  - Extensive ecosystem
  
  **Artifact**: `com.raylon:api:1.0.0`
</Card>

### Language & Runtime

```xml
<properties>
  <java.version>25</java.version>
</properties>
```

<Tip>
  Java 25 provides latest language features including pattern matching, records, and sealed classes.
</Tip>

### Spring Modules

<AccordionGroup>
  <Accordion title="Spring Data JPA">
    **Module**: `spring-boot-starter-data-jpa`
    
    **Features**:
    - Repository abstraction layer
    - Automatic query generation
    - JPA/Hibernate integration
    - Transaction management
    - Entity relationship mapping
    
    **Usage**: Database access layer with `@Repository` interfaces
  </Accordion>
  
  <Accordion title="Spring Web MVC">
    **Module**: `spring-boot-starter-webmvc`
    
    **Features**:
    - RESTful API controllers
    - Request/response handling
    - JSON serialization (Jackson)
    - Exception handling
    - CORS configuration
    
    **Usage**: REST endpoints with `@RestController` and `@RequestMapping`
  </Accordion>
</AccordionGroup>

### Database Driver

| Dependency | Scope | Description |
|------------|-------|-------------|
| **PostgreSQL Driver** | `runtime` | JDBC driver for PostgreSQL database connectivity |

<CodeGroup>
```xml pom.xml
<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <scope>runtime</scope>
</dependency>
```
</CodeGroup>

### Development Libraries

<CardGroup cols={2}>
  <Card title="Lombok" icon="code">
    **Scope**: Optional
    
    Reduces boilerplate code with annotations:
    - `@Getter` / `@Setter` - Auto-generate accessors
    - `@Data` - Combines common annotations
    - `@Builder` - Builder pattern
    - `@Slf4j` - Logger injection
    
    See usage in entity models at `api/src/main/java/com/raylon/api/model/`
  </Card>
  
  <Card title="SpringDoc OpenAPI" icon="book">
    **Version**: 2.6.0
    
    Auto-generates OpenAPI 3.0 documentation:
    - Interactive Swagger UI
    - API schema generation
    - Request/response examples
    
    **Access**: `http://localhost:8080/swagger-ui.html`
  </Card>
</CardGroup>

### Build System

<Tabs>
  <Tab title="Maven">
    **Build Tool**: Apache Maven
    
    **Key Plugins**:
    - `maven-compiler-plugin` - Java compilation with Lombok annotation processing
    - `spring-boot-maven-plugin` - Executable JAR packaging
    
    **Build Output**: `api.jar` (final name configured)
  </Tab>
  
  <Tab title="Lombok Configuration">
    ```xml
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <configuration>
        <annotationProcessorPaths>
          <path>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
          </path>
        </annotationProcessorPaths>
      </configuration>
    </plugin>
    ```
  </Tab>
</Tabs>

## Database Technology Stack

<Card title="PostgreSQL" icon="database">
  **Version**: 15
  
  Advanced open-source relational database:
  - ACID compliance
  - Complex queries and joins
  - JSON/JSONB support
  - Full-text search
  - Robust indexing
  
  **Deployment**: Docker container on port 5432 with persistent volume
</Card>

### Database Features Used

<AccordionGroup>
  <Accordion title="Data Types">
    - `BIGSERIAL` - Auto-incrementing primary keys
    - `VARCHAR` - String fields (id, description)
    - `NUMERIC` / `DOUBLE PRECISION` - Financial amounts
    - `TIMESTAMP` - Date and time records
  </Accordion>
  
  <Accordion title="Constraints">
    - Primary keys on all tables
    - Foreign keys for referential integrity
    - Not null constraints on required fields
  </Accordion>
  
  <Accordion title="Indexes">
    - Primary key indexes (automatic)
    - Foreign key indexes for join performance
  </Accordion>
</AccordionGroup>
