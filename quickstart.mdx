# Quick Start Guide

This guide will help you get EasyACT running and record your first transaction in just a few minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- [Docker](https://www.docker.com/) (version 20.10 or later)
- [Docker Compose](https://docs.docker.com/compose/) (version 2.0 or later)
- Git (for cloning the repository)

<Note>
Docker Desktop includes Docker Compose, so if you have Docker Desktop installed, you're all set!
</Note>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Open your terminal and clone the EasyACT repository:

    ```bash
    git clone https://github.com/RL-Small-Project/EasyACT.git
    cd EasyACT
    ```
  </Step>

  <Step title="Start the Services">
    Launch all services with Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This command will:
    - Build the frontend and backend containers
    - Pull the PostgreSQL 15 database image
    - Start all three services in the background

    <Note>
    The first build may take a few minutes as Docker downloads base images and builds your application.
    </Note>
  </Step>

  <Step title="Verify the Services">
    Check that all containers are running:

    ```bash
    docker-compose ps
    ```

    You should see three services running:
    - `easy_act-web` on port 8000
    - `easy_act-api` on port 8080
    - `easy_act-db` on port 5432
  </Step>

  <Step title="Access the Application">
    Open your web browser and navigate to:

    ```
    http://localhost:8000
    ```

    You should see the EasyACT dashboard!
  </Step>
</Steps>

## Recording Your First Transaction

Now that EasyACT is running, let's record your first transaction:

<Steps>
  <Step title="Navigate to Add Transaction">
    From the dashboard, click on the **Add** or **New Transaction** button to open the transaction form.
  </Step>

  <Step title="Fill in Transaction Details">
    Enter the following information:
    - **Type**: Choose Income or Expense
    - **Amount**: Enter the transaction amount
    - **Category**: Select or create a category (e.g., "Groceries", "Salary", "Transportation")
    - **Date**: Select the transaction date
    - **Description**: (Optional) Add notes about the transaction
  </Step>

  <Step title="Save the Transaction">
    Click **Save** or **Submit** to record your transaction.

    <Note>
    Your transaction is now saved in the PostgreSQL database and will appear in your transaction list and analytics.
    </Note>
  </Step>

  <Step title="View Your Transaction">
    Navigate to the **List View** to see all your transactions, or check the **Chart View** to see visual analytics of your spending patterns.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container fails to start">
    Check the logs for the specific container:

    ```bash
    docker-compose logs web
    docker-compose logs api
    docker-compose logs db
    ```

    Common issues:
    - Port conflicts: Ensure ports 8000, 8080, and 5432 are not in use
    - Insufficient resources: Docker needs adequate memory and CPU allocation
  </Accordion>

  <Accordion title="Cannot connect to the database">
    Verify the database container is running:

    ```bash
    docker-compose ps db
    ```

    If it's not running, restart the services:

    ```bash
    docker-compose restart
    ```
  </Accordion>

  <Accordion title="Frontend shows API errors">
    Ensure the API service is healthy:

    ```bash
    curl http://localhost:8080/swagger-ui.html
    ```

    Check API logs:

    ```bash
    docker-compose logs api
    ```
  </Accordion>

  <Accordion title="Need to reset everything">
    Stop and remove all containers, networks, and volumes:

    ```bash
    docker-compose down -v
    ```

    <Warning>
    This will delete all your data! Only use this if you want to start fresh.
    </Warning>

    Then start again:

    ```bash
    docker-compose up -d
    ```
  </Accordion>
</AccordionGroup>

## Stopping EasyACT

When you're done using EasyACT:

```bash
# Stop containers but keep data
docker-compose stop

# Stop and remove containers (data persists)
docker-compose down

# Stop and remove everything including data
docker-compose down -v
```

<Note>
Your transaction data is stored in the `./data` directory and will persist even after stopping containers, unless you use the `-v` flag.
</Note>
